    启动时提示:没有合适的许可证来实现XXX请求

    百度搜索后,发现可能是因为netapi32.dll破解文件没有正确复制.但我之前刚破解的时候已经复制了.昨天发现windows defender 提示有木马,并且已经隔离威胁.因此,本次报错可能 windows defender 删除了netapi32.dll导致.

    解决方法:重新复制netapi32.dll文件到相应目录中.

    win10远程桌面连接报错:出现身份验证错误,要求的函数不受支持(CredSSP)

    解决 1. win+R,打开运行窗口,输入regedit,打开注册表编辑器,

    2.修改注册表 + 找到路径:计算机\HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System + 在System文件夹内创建文件夹项:\CredSSP\Parameters + 在Parameters文件夹内,新建 DWORD(32)位值(D),文件名为 AllowEncryptionOracle,值为2

    1. 重新远程连接一下,即可正常连接.

    abaqus 提交job时报错:THREE FACTORIZATIONS IN A ROW FAILED

    报错内容: Error in job Job-1: THREE FACTORIZATIONS IN A ROW FAILED. PLEASE CHECK THE MODEL. IT IS POSSIBLE THAT THE MODEL CONTAINS THE KINEMATIC COUPLING DEFINITION SET UP IN A WAY THAT A DEGREE OF FREEDOM HAS NEITHER MASS NOR STIFFNESS

    软件版本: abaqus2023 模型设置: - 采用CFRP+压电材料, 采用三维模型. - cfrp单层尺寸10 * 25 * 0.15 mm,pvdf膜尺寸:10 * 25 * 0.052 mm. - 复材铺层角度: 90/0/90,pvdf极化方向为厚度方向. - 约束条件:悬臂板约束

    首次提交: 报错信息:THREE FACTORIZATIONS IN A ROW FAILED

    解决过程: 1. 检查约束条件/单元类型/压电极化方向是否正确.[√] 2. 检查网格尺寸(有up说单元最大边和最小边的比值太大也会导致这个error)[√] 3. 单元共节点,自由度传递没问题[√] 4. 最后发现:压电单元节点自由度9为电势(electric potential).在eigenfrequency Extract step中,如果用到了压电单元,则必须至少约束住一个节点的电势自由度.

    最后将压电单元的节点电势自由度约束住,0电势边界条件,提交job成功.

    冲突的原因,网上很多都说是因为UG和ABAQUS的许可证冲突了.

    我电脑上先是装了ABAQUS2020,又装了CATIA2016,还附装了一个Fibersim for CATIA.之后又装了UG10,UG10的许可证破解我没有采用传统的破解方法.而是采用了优胜万能许可证.CATIA的许可证经常就报错.

    解决方法:修改UG许可证端口号

    1. 找到UG的许可管理工具Lmtools,管理员身份打开

    1. 找到许可证文件的路径

    1. 修改splm8.lic文件中的27800为27801

    1. 相应地,在环境变量_系统变量中,修改所有与UG有关的27800为27801

    1. 打开lmtools中重新读取许可文件爱你/stop /start 服务.最后显示开启服务成功即可

    abaqus接触分析不收敛: Displacement increment for contact is too big

    1. 检查接触属性,接触面定义是否正确.
    2. 初始分析步,载荷调小试试.
    3. 接触面网格尽量一致,不要差距过大,否则长时间找不到节点

    4. 单元类型尽量用一阶

    5. ABAQUS收敛控制经验谈（一）——漫谈不收敛的原因

    接触分析警告: 存在2个未联通区域

    网友经验: CH3.1.1 Using the Abaqus environment settings 提到 abaqus_V6.env 加入 Unconnected_regions=ON 在job中，用datacheck计算完毕后进入后处理，没有连接在一起的区域，会自动分成若干element groups
